CVODE Trajectory Generator Documentation

Introduction

The CVODE Trajectory Generator is a command-line program based on the CVODE library which integrates a set of differential equations to calculate a trajectory in phase space. The initial conditions for the system are given on the command line and the differential equations are provided by a "problem" object. The supported problem types are listed at the end of this section.

The Generator is capable of integrating the differential equations for only one set of initial conditions per execution. The initial conditions are specified on the command-line, either by the –initial-vector option or by options specific to a problem. The format of output data may vary with the problem being used, but all current problems output columns of data (separated by tabs) representing the state of the system at many intermediate points during the integration.

The Escape problem: an example

The Generator was originally written for the Escape problem. In this problem, we simulate the path taken by a small particle of infinitesimal mass under the Newtonian gravitational influence of two massive bodies which orbit each other in concentric circles. The problem is a restricted version of the classical three-body problem; it is implemented in the RestrictedThree class.

In the current build system, escape and rthree are identical executables which integrate the RestrictedThree problem. With the default parameters and initial conditions, escape produces output like this:

 $ ./escape
#t      x       y
0       0.8     0
0.0321  0.867814        0.0224674
0.0641  0.923033        0.0448505
0.0961  0.97147 0.0672085
0.1281  1.01532 0.0895327
0.1602  1.05587 0.111886
0.1923  1.09369 0.134191
0.2244  1.12926 0.156445
0.2565  1.16294 0.178643
...

The first column is the system time, where is the start of integration, and marks the time when the 2-body system completes its first revolution. The next two columns give the familiar Cartesian coordinates of the projectile's position in the plane. When additional options are specified, additional columns may appear:

 $ ./escape -E -G -X -0.24 -Y -0.584 
#t      x       y       e
0       0.661989        -0.092494       0
0.0321  0.638866        -0.1253 8.50784e-11
0.0641  0.619952        -0.145911       1.07883e-10
0.0961  0.603042        -0.159546       1.13102e-10
0.1281  0.587256        -0.16827        1.22007e-10
0.1602  0.572063        -0.173198       1.23826e-10
0.1923  0.557199        -0.174981       1.24331e-10
0.2244  0.542441        -0.174081       1.24602e-10
0.2565  0.527617        -0.170828       1.2514e-10
...

Here the initial conditions are specified by the -X and -Y options. The -G option specifies how the -X and -Y options are to be interpreted, and the -E option adds an additional column giving the deviation from initial pseudoenergy. (all these options are documented in RestrictedThree::getOptionString())

The frequency of output is controlled with the -N option and the upper bound of integration is specified with -e. For example: 

 $ ./escape -e 44 -N 4
#t      x       y
0       0.8     0
11.0001 2.28949 3.84313
22.0002 -0.592399       0.147827
33.0002 -4.77448        0.0824081
44      -3.82037        -1.39882
 $

The escape executable and others like it are infrequently used alone like this. Usually they are used in the so-called "dispatch" scripts developed for the Escape problem, which can be used to run escape millions of times with different intial conditions and distribute the computational task across many computers using Apple's Xgrid software.

Command-line parameters

The generator supports many command-line parameters. The following parameters are valid for all problems:

  • -p:
    Select problem to simulate. This is currently not supported; for now each problem produces its own executable.
  • -I, –integration-method:
    Select integrator. This should be either "bdf" or "adams".
    • CVODE Manual: 5.5.1 CVODE initialization and deallocation functions - CVodeCreate()
  • -i, –iterator: Select iterator. This should be either "newton" or "functional".
    • CVODE Manual: 5.5.1 CVODE initialization and deallocation functions - CVodeCreate()
  • -v, –initial-vector:
    A vector of initial conditions, separated by commas. The list must be of the length required by the problem in use.
  • -r, –relative-tolerance:
    Relative tolerance. Values around 1e-8 or 1e-12 have been found to work well.
    • CVODE Manual: 5.5.2 Advice on choice and use of tolerances
  • -T:
    Absolute tolerances, as a comma-separated list. The list must be of the length required by the problem in use.
    • CVODE Manual: 5.5.2 Advice on choice and use of tolerances
  • -t, –absolute-tolerance:
    Absolute tolerance, as a scalar. The list of absolute tolerances is filled with copies of this value.
    • CVODE Manual: 5.5.2 Advice on choice and use of tolerances
  • -b, –begin:
    Beginning time. The unit of time is defined by the problem in use. It is generally safe to leave this at its default; in fact, the generator may have trouble if this parameter is used. Default: 0
  • -e, –end:
    Ending time.
  • -s, –timestep:
    Internal timestep for CVODE integration. Smaller values allow the integrator to go farther without dying with a "too much work" error. But make this too small, and you gain no precision but lose performance.
  • -N, –number-points, –data-length:
    Number of points to record. Default: 1000
  • -L, –logtime-start:
    Enable logarithmic time for data recording and set .
  • -n, –logtime-n:
    Enable logarithmic time for data recording and set so
  • -m, –max-internal-steps:
    Maximum number of internal CVODE steps to take per timestep. Bigger values for this parameter tend to eliminate "too much work" errors and seem to have no negative effect on performance. Default: 1000000

Other options are specified by the particular problem in use.

  • SingleBody::getOptionString()
  • TwoBody::getOptionString()
